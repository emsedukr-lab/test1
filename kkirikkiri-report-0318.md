## 1. 프로젝트 개요

### 1.1 목표

사진 또는 PDF에 포함된 **한글 필기체**를 자동으로 인식(OCR)하고, 인식된 데이터를 지정된 **웹사이트에 자동 입력**하는 통합 도구를 기획한다. 사용자가 수작업으로 수행하던 "종이 문서 읽기 → 웹 시스템 입력" 과정을 자동화하여 시간과 오류를 대폭 줄이는 것이 핵심 가치이다.

### 1.2 용도

| 용도 | 설명 | 예시 |
|------|------|------|
| **시험/채점** | 학생 답안지 필기를 인식하여 성적 시스템에 자동 입력 | 대학 성적 입력, 수능 채점 보조, 단답형 답안 처리 |
| **업무/서류 자동화** | 양식 내 필기 필드를 인식하여 사내 시스템/정부 사이트에 입력 | 계약서, 신청서, 접수 양식 디지털화 |
| **설문/조사 데이터** | 체크박스 + 주관식 응답을 인식하여 데이터베이스/폼에 입력 | 설문 결과 집계, 투표 집계, 만족도 조사 |

### 1.3 타겟 사용자

- **교육기관**: 교사, 교수, 채점 보조 인력
- **공공기관/기업**: 행정 담당자, 서류 처리 업무자
- **연구기관**: 설문 분석 연구원, 데이터 입력 담당자
- **개인 사업자**: 수기 서류가 많은 소규모 사업체

### 1.4 대상 사이트

미정(범용 설계). 특정 사이트에 종속되지 않고, 설정 파일 기반으로 다양한 웹사이트에 대응하는 구조로 설계한다.

```
        ▼
┌───────────────────┐     ┌───────────────────┐     ┌───────────────────┐

                          ┌───────────────────┐     ┌───────────────────┐

                          │ - 스크린샷 저장    │     │ - 필드 매핑       │
                          │ - 성공/실패 리포트  │     │   (설정파일/AI)    │
                          │ - 실패 건 재처리    │     │ - 데이터 입력     │
                          └───────────────────┘     │ - 저장/제출       │
                                                    └───────────────────┘
```

### 2.2 주요 모듈 상세

| 모듈 | 역할 | 핵심 기술 |
|------|------|----------|
| **이미지 전처리** | 입력 이미지 품질 개선, 인식 영역 추출 | OpenCV, pdf2image, 기울기 보정, 이진화 |
| **OCR 인식 엔진** | 필기체 텍스트를 문자열로 변환 | Clova OCR, PaddleOCR, LLM Vision |
| **데이터 구조화** | OCR 결과를 필드별 구조화 데이터로 변환 | 정규식, 후처리 로직, 타입 검증 |
| **웹 자동 입력** | 구조화 데이터를 웹사이트 필드에 자동 입력 | Playwright, browser-use |
| **결과 검증** | 입력 정확성 확인 및 리포트 생성 | 스크린샷 비교, 값 대조, 로그 기록 |

## 3. 기술 스택 추천

### 3.1 OCR 기술 스택

#### 1순위: Naver Clova OCR (Template OCR)

| 항목 | 내용 |
|------|------|
| **한글 필기체 정확도** | 75~90% (클라우드 서비스 중 최고) |
| **핵심 강점** | Template OCR로 양식 영역 시각적 지정 (최대 50개 필드) |
| **근거** | ICDAR 2019 국제 문서 인식 대회 4개 부문 1위, 한국어 특화 학습 데이터 보유 |
| **비용** | 유료 (호출 수 기반, 플랜별 차등) |
| **적합 용도** | 시험/채점 (답안지 영역 지정), 업무/서류 (양식 필드 인식) |

**선정 이유**: 한글 필기체 인식에서 압도적 1위. Template OCR 기능은 정해진 양식(답안지, 신청서 등)의 특정 칸을 지정하여 인식하는 본 프로젝트의 핵심 요구사항에 정확히 부합한다.

#### 2순위: Google Cloud Vision API

| 항목 | 내용 |
|------|------|
| **한글 필기체 정확도** | 70~85% |
| **핵심 강점** | DOCUMENT_TEXT_DETECTION으로 밀집 필기 문서 처리, 문서 구조 상세 반환 |
| **근거** | 글로벌 인프라 안정성, 풍부한 SDK/문서, 무료 1,000건/월 |
| **비용** | 무료 1,000건/월, 이후 $1.50/1,000건 |
| **적합 용도** | 다국어 혼합 문서, 글로벌 서비스 확장 시 |

**선정 이유**: Clova OCR 대비 글로벌 범용성과 안정성이 뛰어나며, 무료 티어로 소규모 PoC에 적합하다. 영어+한글 혼합 문서 처리에도 강하다.

#### 무료 대안: PaddleOCR PP-OCRv5

| 항목 | 내용 |
|------|------|
| **한글 필기체 정확도** | 55~70% (오픈소스 중 최고) |
| **핵심 강점** | 완전 무료, 오프라인 실행, PDF 직접 처리, 배치 최적화 |
| **근거** | PP-OCRv5에서 필기체 대폭 개선, 106개 언어 지원, 3단계 파이프라인 완비 |
| **비용** | 무료 (서버 인프라 비용만 발생, GPU 서버 시 월 $50~100) |
| **적합 용도** | 비용 제한 환경, 데이터 외부 유출 불가 환경, 대량 배치 처리 |

**선정 이유**: 개인정보가 포함된 서류를 외부 클라우드에 전송할 수 없는 환경에서 유일한 실용적 대안이다. 클라우드 API 대비 정확도는 낮지만, 비용이 0이며 커스텀 학습으로 개선 가능하다.

#### 보조: LLM Vision 모델 (GPT-4o / Claude / Gemini)

| 항목 | 내용 |
|------|------|
| **필기 인식 정확도** | 95~99% (영어 벤치마크 기준, 한글은 다소 낮을 수 있음) |
| **활용 방식** | 1차 OCR 결과 중 저신뢰(confidence < 0.7) 건만 2차 검증 |
| **근거** | 기존 OCR 평균 64% 대비 99%+ 달성, 컨텍스트 이해로 오타/약어 보정 가능 |
| **비용** | 이미지당 $0.01~0.05 (전체 적용 시 비용 과다, 선별 적용 권장) |

**선정 이유**: 하이브리드 파이프라인의 2차 검증 단계로 활용하면 비용 대비 정확도를 극대화할 수 있다.

#### 특수 용도: Azure AI Document Intelligence

| 항목 | 내용 |
|------|------|
| **핵심 강점** | 체크박스/선택마크(selectionMark) 자동 인식, Custom 모델 학습 |
| **적합 용도** | 설문/조사 (체크박스 + 주관식 혼합 양식) |
| **비용** | Read $1.50/1K페이지, Layout $10/1K페이지 |

#### 제외 대상

| 기술 | 제외 사유 |
|------|----------|
| **AWS Textract** | 한국어 미지원 (영/스페인/독/이탈/불/포르투갈어만) |
| **Tesseract** | 한글 필기체 30~50%, 실용성 부족 |
| **Kakao OCR** | 베타 성격, 필기체 특화 기능 없음, Clova 대비 열세 |
| **TrOCR/Donut** | 한글 pre-trained 모델 없음, fine-tuning에 ML 전문인력+데이터셋 필요 |

### 3.2 웹 자동화 기술 스택

#### 1순위: Playwright (Python)

| 항목 | 내용 |
|------|------|
| **속도** | 가장 빠름 (<100ms/액션, WebSocket 기반) |
| **핵심 강점** | auto-wait 내장, 세션 저장/복원(storage_state), codegen 녹화, 크로스 브라우저 |
| **근거** | 2026년 기준 신규 프로젝트 표준, Chromium/Firefox/WebKit 모두 지원 |
| **비용** | 무료 |
| **적합 용도** | 모든 시나리오의 기본 브라우저 자동화 엔진 |

**선정 이유**: 속도, 안정성, 세션 관리, 다중 브라우저 지원 모든 면에서 최고. OCR 데이터 자동 입력의 핵심 엔진으로 최적이다.

#### 2순위: browser-use (AI 하이브리드)

| 항목 | 내용 |
|------|------|
| **GitHub Stars** | 78,000+ (2026년 기준) |
| **핵심 강점** | LLM으로 브라우저 완전 제어, 셀렉터 불필요, 자연어 태스크 지시 |
| **근거** | WebVoyager 벤치마크 89.1%, 월 파손율 <5% (전통 RPA 15~25% 대비) |
| **비용** | LLM API 비용 ($0.02~$0.30/액션) |
| **적합 용도** | 초기 필드 매핑 자동화, 새로운/변경된 사이트 대응 |

**선정 이유**: 사이트마다 셀렉터를 수동으로 설정하는 부담을 AI가 해결한다. 초기 매핑 → 설정 저장 → 이후 Playwright 실행의 하이브리드 패턴에 최적.

#### 제외 대상

| 기술 | 제외 사유 |
|------|----------|
| **Selenium** | Playwright 대비 느리고, auto-wait 없어 코드 장황, SPA 대응 열세 |
| **Puppeteer** | Chromium 전용 + Node.js 전용, Playwright의 하위호환 |
| **Cypress** | E2E 테스트 전용, 크로스 도메인 제한, 범용 자동화 부적합 |
| **UiPath/AA** | 연간 수백만원, 셀렉터 의존 유지보수 부담, "RPA 30~50% 첫 해 실패" |

## 4. 용도별 최적 구성

### 4.1 시험/채점

```
[답안지 스캔] → [Clova Template OCR] → [학번/이름/답/점수 추출]
     → [저신뢰 건 LLM 재검증] → [Playwright + 설정파일 매핑]
     → [성적 시스템 자동 입력] → [입력값 대조 검증]
```

| 항목 | 내용 |
|------|------|
| **OCR** | 1순위: Clova Template OCR, 2순위: Google Cloud Vision, 무료: PaddleOCR + 영역 크롭 |
| **웹 자동화** | Playwright + 설정 파일 매핑 (반복 입력 최적화) |
| **예상 정확도** | OCR 75~90% + LLM 검증 시 90~95% |
| **비용 (월 1,000건)** | Clova OCR 유료 플랜 + Playwright 무료 = 추정 월 $20~50 |
| **핵심 요구** | 높은 정확도 (성적 오류 불허), 배치 처리, 검증 필수 |

**주의사항:**
- OMR(마킹) 영역은 별도 OMR 전용 도구로 처리
- 수식/특수 기호는 일반 OCR로 인식 어려움 (수학 OCR 별도 모델 필요)
- 입력 기간 제한이 있으므로 안정성이 속도보다 중요

### 4.2 업무/서류 자동화

```
[서류 스캔] → [Clova Template OCR / Azure Custom 모델]
     → [key-value 쌍 추출 + 타입 검증]
     → [browser-use AI 매핑 (초기) / Playwright 설정 매핑 (반복)]
     → [사내 시스템/정부 사이트 입력] → [결과 검증]
```

| 항목 | 내용 |
|------|------|
| **OCR** | 1순위: Clova Template OCR, 2순위: Azure Doc Intelligence (Custom 모델), 무료: PaddleOCR/Donut |
| **웹 자동화** | browser-use/Skyvern (다양한 양식 대응) → 매핑 저장 후 Playwright 전환 |
| **예상 정확도** | OCR 70~85% + 후처리 시 85~92% |
| **비용 (월 1,000건)** | Clova OCR + browser-use 초기 매핑 = 추정 월 $25~65 |
| **핵심 요구** | 다양한 양식 대응, 복잡한 인증 처리, 보안/프라이버시 |

**주의사항:**
- 정부 사이트는 인증서/보안 모듈(ActiveX 등) 이슈 다수
- 개인정보 포함 서류는 온프레미스 솔루션(PaddleOCR) 선호
- 사이트 변경 빈번 → AI 자동 매핑의 가치가 높음

### 4.3 설문/조사 데이터

```
[설문지 스캔] → [Azure Doc Intelligence (체크박스)] + [Clova OCR (주관식)]
     → [체크 상태 판별 + 텍스트 인식 + 구조화]
     → [API 직접 호출 (구글폼) / Playwright (기타 폼)]
     → [결과 집계/통계]
```

| 항목 | 내용 |
|------|------|
| **OCR** | 1순위: Azure Doc Intelligence (selectionMark 체크박스 인식), 2순위: Clova Template OCR, 무료: PaddleOCR + OpenCV |
| **웹 자동화** | 1순위: API 직접 호출 (Google Forms API 등), 2순위: Playwright + 설정 파일 |
| **예상 정확도** | 체크박스 90~95%, 주관식 OCR 70~85% |
| **비용 (월 1,000건)** | Azure Read $1.50 + API 무료 = 추정 월 $5~25 |
| **핵심 요구** | 체크박스 + 텍스트 혼합 인식, 대량 배치, 통계 출력 |

**주의사항:**
- 구글폼은 Google Forms API로 직접 제출 가능 (브라우저 자동화 불필요)
- 체크 마크 스타일 다양성(V, X, 색칠 등) 대응 필요
- 대량 제출 시 스팸 방지 속도 제한 준수

## 5. 핵심 기능 상세

### 5.1 이미지/PDF 업로드 + 영역 지정

**기능:**
- 이미지(JPEG, PNG, BMP, TIFF) 및 PDF 파일 업로드
- 드래그 앤 드롭 또는 파일 탐색기를 통한 업로드
- PDF의 경우 페이지별 자동 분리 및 이미지 변환
- 사용자가 인식할 영역을 시각적으로 드래그하여 지정
- 양식 템플릿 저장: 같은 양식의 영역 설정을 재사용

**구현 방안:**
- 프론트엔드: Canvas 기반 영역 지정 UI (fabric.js 또는 react-konva)
- 백엔드: pdf2image로 PDF→이미지 변환, OpenCV로 전처리
- Clova Template OCR 연동 시 Naver 웹 빌더의 영역 지정 기능 활용 가능

### 5.2 OCR 인식 + 신뢰도 표시

**기능:**
- 지정 영역별 텍스트 인식 수행
- 각 인식 결과에 **신뢰도 점수**(0~1.0) 표시
- 신뢰도 기준:
  - 0.9 이상: 녹색 (자동 확정)
  - 0.7~0.9: 노란색 (사용자 확인 권장)
  - 0.7 미만: 빨간색 (사용자 수정 필수) → LLM 2차 검증 자동 트리거
- 인식 결과를 테이블 형태로 표시하여 사용자가 확인/수정 가능
- 원본 이미지와 인식 결과를 나란히 비교 표시

**하이브리드 파이프라인:**
```
[이미지] → [1차 OCR (Clova/PaddleOCR)]
               │
               ├─ 신뢰도 >= 0.7 → 결과 확정
               │
               └─ 신뢰도 < 0.7 → [2차 LLM Vision 검증]
                                       │
                                       └─ 보정된 결과 반환
```

### 5.3 웹사이트 필드 매핑 (자동 + 수동)

**자동 매핑 (AI 기반):**
1. browser-use가 대상 웹페이지의 DOM/스크린샷 분석
2. input 필드의 label, placeholder, aria-label, name 속성 수집
3. OCR 데이터의 키(이름, 학번, 점수 등)와 웹 필드를 자동 매칭
4. 매칭 결과를 사용자에게 시각적으로 표시하여 확인 요청

**수동 매핑:**
- 사용자가 OCR 필드와 웹 필드를 드래그 앤 드롭으로 연결
- CSS 셀렉터 또는 XPath를 직접 입력하여 매핑
- 매핑 테스트 기능: 샘플 데이터로 실제 입력 미리보기

**매핑 저장 (YAML 설정 파일):**
```yaml
site: "학교성적시스템"
url: "https://school.example.com/grades"
login:
  url: "https://school.example.com/login"
  username_field: "#user-id"
  password_field: "#password"
  submit: "button[type='submit']"
fields:
  - ocr_key: "이름"
    selector: "#student-name"
    type: "text"
  - ocr_key: "학번"
    selector: "#student-id"
    type: "text"
  - ocr_key: "점수"
    selector: "input[name='score']"
    type: "number"
    validation:
      min: 0
      max: 100
submit_button: "#save-btn"
success_indicator: ".alert-success"
```

**하이브리드 전략:**
1. 첫 실행 → AI 자동 매핑 → 사용자 확인 → 설정 파일 저장
2. 이후 실행 → 저장된 설정 파일 사용 (AI 비용 절감, 속도 향상)
3. 매핑 실패 시 → AI 재매핑 시도 (자동 복구)

### 5.4 자동 입력 + 결과 확인

**자동 입력 프로세스:**
1. Playwright로 대상 사이트 접속 (저장된 세션 복원 또는 로그인)
2. 설정 파일 기반으로 각 필드에 데이터 입력
3. 입력 후 저장/제출 버튼 클릭
4. 성공 메시지 또는 에러 확인

**결과 확인 (검증):**
- **값 대조 검증**: 입력 후 필드 값을 다시 읽어 OCR 데이터와 비교
- **시각적 확인**: 입력 전후 스크린샷 자동 저장
- **HTTP 응답 확인**: 저장 API 응답 코드(200 OK) 확인
- **성공 지표 확인**: 성공 메시지 요소(.alert-success 등) 표시 여부

**에러 복구:**
- 네트워크 타임아웃 → 최대 3회 재시도
- 요소 미발견 → 대기 후 재시도 / 대체 셀렉터
- 세션 만료 → 자동 재로그인
- CAPTCHA → 사용자 개입 요청 알림

### 5.5 배치 처리

**기능:**
- 여러 건의 이미지/PDF를 한번에 업로드하여 순차 처리
- 처리 진행률 실시간 표시 (현재 N/전체 M건)
- 실패 건 자동 스킵 + 실패 목록 별도 저장
- 처리 완료 후 결과 리포트 생성 (성공/실패/스킵 건수)
- 중간 체크포인트 저장: 중단 후 이어서 처리 가능

**배치 처리 흐름:**
```
[이미지 목록] → [순차 OCR 처리] → [구조화 데이터 목록]
     → [사이트 로그인 (1회)] → [반복: 데이터 입력 → 검증 → 다음 건]
     → [결과 리포트 생성]
```
